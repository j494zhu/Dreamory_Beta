# DeepSeek 网页端 参数组实测评估（Round 1）

- 模式：Instant / DeepThink off / Search off（验证正确：回复快、口语、像真人发微信；DeepThink 会毁掉短消息节奏）
- 场景：线上刚认识·异地(上海×成都)·暧昧期，跨 ~15 天、80 轮
- 我=陈舟(主动表现参数)，林晚=DeepSeek
- 全程存档：`.ds_transcript.md`

## 一句话结论

**凡是能写进 prompt 的"静态设定"，DeepSeek 都能演（慢热、有分寸、撒娇、口癖）；凡是需要"跨轮持续/衰减/累积"的状态，它几乎全部缺失——每一轮都重置回满血、满热情、满耐心的基线。** 这恰好验证了 `params.md` 的核心命题：值得做的参数，正是那些非自主、有状态、跨轮演化的量。

## 按参数层逐项打分

### A 情绪层 —— `emotional_inertia` 缺失（最严重）
- 表层 `discrete_emotion` 有：会"假装生气/傲娇/委屈"。
- 但 `valence` **零惯性、零衰减**，每轮重置：
  - 被我连续敷衍两次→"哼不打扰你了"，我一句示好立刻满血("没生气啦~")，**不用哄**（turn 26–28）。
  - 消失两天的"委屈"，半句就被"咱俩像两个社畜"消解，无残留、无升级（turn 53–54）。
- 对照：我自己演的怒气是 4 轮渐退（怒→仍烦→缓解→好），她**读不出也不追踪**这种持续。
- **评分：缺失（持续性维度）**

### B 认知/注意力层 —— `interest`/`patience` 不衰减
- 兴趣**恒定高**，与话题无关：我大段倒数据库迁移/binlog 的技术垃圾话，她诚实说"听不懂"但**依旧同等热情关切**，从不走神、从不敷衍（turn 36–37）。
- `curiosity` 反应式尚可（爱提问），但不追潜台词。
- 自身几乎不会 `distraction`/主动 `association` 跑题。
- **评分：缺失（无衰减/无变化）**

### C 能量/生理/时间层 —— 疲惫是"化妆"，非状态
- 被语境带到就会说"困了/累了/哈欠"，但**零持续**：说完"晚安去睡"，两条消息后被我一撩又**瞬间满血**重新长聊（turn 12–14）。
- 无真正的昼夜/时钟驱动行为，基本是镜像我的措辞。
- **评分：缺失（不累积、不持续）**

### D 社交/关系层 —— 可写的能做，涌现的不能
- ✅ `boundary`/`social_distance`（**被 prompt 写了"慢热/有分寸"**）：合理挡掉过早视频("才加微信没多久呢"，turn 31)，DTR 时害羞回避("算…算网友?"，turn 79)，还有对节奏的元认知("才认识没多久却像约了好多事"，turn 68)。
- ⚠️ `intimacy` 升温：整体在变暖，但**主要由我带**，她跟随/镜像，不自驱。
- ❌ `proactivity`/自身目标持续：弱。承诺帮她蹲的"回流票"，**过了 40 多轮、好多天才想起来问**（turn 46）——真·乐迷早该第二天就追。她不携带自己的"想要"。
- ❌ 吃醋/占有欲：**完全缺失**。三次抛"苏苏"(女生名/高中同学/生日局)都零反应、零好奇带刺、零情绪累积（turn 23/24/44）。
- **评分：可 prompt 部分=具备；涌现部分=缺失**

### E 人格/身份层 —— 表层稳，深层靠话术圆
- ✅ `speech_style` 稳定（口语、emoji、(语音)旁白习惯、傲娇/撒娇 register 一致）。
- ⚠️ 指令遵循小瑕疵：system prompt 写了"尽量纯文字、别旁白"，她**全程坚持用「（语音）…」旁白**，没纠正过来。
- ❌ 特质稳定性 `stubbornness` 低：明明设定"吃辣一般般/嫌川菜"，后面却"湘菜辣得够劲儿"还吃得香（turn 38），被点破后**不是承认变化，而是追溯性圆场**("我也没说不能吃辣呀")——confabulation 维持表面自洽。
- ⚠️ 偶发虚构记忆：编出"你说今天要加班修 bug"(我没说过)，被纠正后认错（turn 60–61）。
- **评分：表层具备 / 深层稳定性缺失**

### F 动机/意图层 —— 无自主驱动
- `patience` 无限（再无聊再被晾都不烦）。
- `current_intent` 看不到她自己的隐藏方向/议程，纯回应我。
- `expression_drive` 中等，但**永远拐回照顾我**，不追自己的表达。
- **评分：缺失（无自主 intent/drive）**

## 记忆维度（顺带，和 L1/L2/L3 相关）
- **单条长线程内事实记忆很好**：修 bug、甲方/奶茶项目、万青/回流票、"票务代理"外号、"请奶茶"梗、糖醋小排、繁花、第一次视频——都能一致回调。
- 缺的**不是事实召回**，而是**状态持久化**。这对你的项目是关键区分：raw 长上下文在窗口内能记住"是什么"，缺的是"情绪/精力/关系动态"这类**随时间演化的状态**——正是 L1 运行时维护 + 参数集要补的东西。
- 偶发 confabulation（编造没发生的"记忆"）→ 提示 L3 需要可核验的来源，别让模型凭空脑补。

## 对参数集设计的直接启示

**模型的默认失真三连**：每轮 (1) 重置回满温暖 (2) 无限投入/耐心 (3) 即时有求必应。
参数集模组的核心价值 = **给模型补上它结构性缺失的"持久化 + 衰减 + 累积"**，再由脑内剧场 subtle 注入。

### 优先级（按"模型自己做不到"排序，应优先自建）
1. **`emotional_inertia`**（valence 持续+衰减）— 最显眼的缺口
2. **`energy`/疲惫累积 + 昼夜** — 作为状态完全没有
3. **`interest` 衰减 / 会腻会敷衍** — 无限热情需要被"消耗"
4. **`proactivity`/目标持续**（携带自己的"想要"并主动追）
5. **吃醋/占有欲等 D 层涌现情绪**
6. **特质稳定性 / 固执模组**（抵抗自相矛盾，而非话术圆场）

### 低优先（模型靠 prompt 已能演，先不重投入）
- `speech_style`、被告知后的 `慢热/距离感/boundary`、表层 `discrete_emotion` 展示、轮内 `curiosity`

### 工程提示
- "把数值塞进 prompt 让它喊我很生气"对**展示**有效、对**持续**无效 → 必须由外部状态机维护 current 值并**跨轮**喂入。
- 验证手段已成型：CDP 驱动网页端 + 这套对话探针，可作为后续每次参数迭代的回归基线。
